## OBIETTIVO DEL PROGETTO

Dashboard web per il tracciamento temporale degli eventi di vendita nei negozi Wind3 (franchising).
Permette di seguire la "customer journey" di ogni cliente nel tempo: quali servizi ha attivato, con quale negozio, con quale addetto, e se i contratti sono ancora attivi nei mesi successivi.

### Logica di business fondamentale
- **T0** = prima attivazione MOBILE di un cliente â†’ inizio del rapporto consulenziale
- **T1, T2, T3...** = mesi successivi in cui lo stesso cliente riappare (nuovi prodotti, rinnovi, canoni)
- Il cliente accumula prodotti nel tempo: SIM â†’ fisso â†’ energia â†’ assicurazioni
- L'obiettivo finale Ă¨ costruire un "portafoglio clienti" per ogni addetto del negozio

## LOGICHE DI BUSINESS IMPLEMENTATE

### Riconoscimento T0 (prima attivazione cliente)
Un evento T0 si identifica quando:
- `TIPO_FONIA` = `MOBILE`
- `DT_ATTIVAZIONE` coincide con il `PERIOD` dell'evento (stesso mese)

In codice Python:
```python
is_mob   = df["TIPO_FONIA"] == "MOBILE"
attiv_ym = pd.to_datetime(df["DT_ATTIVAZIONE"], errors="coerce").dt.strftime("%Y%m").fillna("")
df["IS_T0"] = is_mob & (attiv_ym == df["PERIOD"].fillna(""))
```

### PDV (negozio) per contratto
Il PDV di ogni `CODICE_CONTRATTO` = PDV della **prima `DATA_EVENTO` cronologica** per quel contratto:
```python
cc_pdv = (df.sort_values("DATA_EVENTO")
           .groupby(["FISCAL_CODE","CODICE_CONTRATTO"])["PDV"]
           .first())
```

### Negozio principale per cliente (`pdvp`)
Il `pdvp` di ogni cliente = PDV del suo T0 (prima attivazione Mobile). Se T0 assente, PDV del primo evento cronologico:
```python
t0_pdv_s    = df[df["IS_T0"] & (df["PDV"]!="")].sort_values("DATA_EVENTO").groupby("FISCAL_CODE")["PDV"].first()
first_pdv_s = df[df["PDV"]!=""].sort_values("DATA_EVENTO").groupby("FISCAL_CODE")["PDV"].first()
pdvp_s = t0_pdv_s.combine_first(first_pdv_s)
```

### Posizionamento sull'asse temporale
- **MOBILE / ASSICURAZIONI**: posizionati per `DATA_EVENTO` (fallback `DT_ATTIVAZIONE`)
- **FISSO / ENERGIA**: posizionati per `DT_ATTIVAZIONE` (fallback `DATA_EVENTO`) â€” perchĂ© c'Ă¨ sempre un gap tra firma e attivazione effettiva

### Storni

```python
df["IS_STORNO"] = df["ITEM_CATEGORY"].str.contains("Addebito Negativo", case=False, na=False)
```

### Gestione dei file Excel su Replit
```python
# Pattern di caricamento (stesso della fase 1)
import glob, pandas as pd

def load_excel_files(cartella):
    files = sorted(glob.glob(f"{cartella}/*_unificato*.xlsx"))
    files = [f for f in files if not os.path.basename(f).startswith("~$")]
    frames = []
    for path in files:
        xf    = pd.ExcelFile(path)
        sheet = xf.sheet_names[0]  # sempre il primo foglio
        hdr   = pd.read_excel(path, sheet_name=sheet, nrows=0)
        cols  = [c for c in COLS_NEEDED if c in hdr.columns]
        df    = pd.read_excel(path, sheet_name=sheet, usecols=cols, dtype=str)
        frames.append(df)
    return pd.concat(frames, ignore_index=True)
```
